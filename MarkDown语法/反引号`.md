在 Markdown 语法中，反引号（**`**，Backtick，位于键盘 Tab 键上方）是程序员和技术文档撰写者最常用的符号之一。它的核心作用是**标识代码**。

以下是反引号的三种主要用法：

---

### 1. 行内代码 (Inline Code)

当你只想在一段话中间插入一小个变量名、函数或命令时，使用**单个反引号**将其包裹。

- **语法**：`` `代码内容` ``
    
- **效果**：代码会以等宽字体显示，通常伴有淡淡的背景色，使其在文本中脱颖而出。
    
- **示例**：
    
    > 请使用 `printf()` 函数输出结果，并检查 `int` 类型的取值范围。
    

---

### 2. 代码块 (Code Blocks)

如果你需要展示多行代码，则使用**三个反引号**（也称为“围栏代码块”）。

- **语法**：
    
    Plaintext
    
    ````
    ```语言名称
    代码内容
    代码内容
    ````
    
- **功能**：
    
    - **语法高亮**：在开头的三个反引号后指定语言（如 `c`, `python`, `sql`, `json`），Markdown 渲染器会自动根据语法上色。
        
    - **保持格式**：代码块内的空格、缩进、换行都会原样保留。
        

---

### 3. 处理特殊符号

Markdown 具有特定的转义规则，但在反引号内部，所有的 Markdown 标签都会失效。

- **防止误触发**：比如你想在文档里写 `*这个不是斜体*`，如果不用反引号包裹，Markdown 会把它识别为斜体。用了反引号，它就会被视为普通字符。
    
- **嵌套反引号**：如果你想在代码里显示一个反引号，可以用**双反引号**包裹。
    
    - **语法**： ` `` `` -> 效果： `
        

---

### 总结对比

| **符号数量**           | **用法** | **适用场景**                  |
| ------------------ | ------ | ------------------------- |
| **单反引号 (`` ` ``)** | 行内代码   | 变量名、简单命令、路径（如 `/usr/bin`） |
| **三反引号 (` ` `)**   | 多行代码块  | 完整的算法逻辑、配置文件、日志输出         |
| **带语言标识**          | 语法高亮   | 让 C 代码显示关键字高亮，提升阅读体验      |
