## 1. 递归的核心公式

一个有效的递归函数必须包含两个部分，缺一不可：

1. **基准情形（Base Case / 终止条件）**：告诉函数什么时候停止，防止无限循环。
    
2. **递归步骤（Recursive Step）**：函数调用自身，并且每次调用都必须让问题变得更“小”，向基准情形靠近。
    

> **比喻**：你在排队，想知道自己是第几排。你问前面的人，前面的人又问他前面的人（递归步骤）。直到问到第一排的人，他说“我是第一排”（基准情形），然后答案再一级一级传回来。

---

## 2. 递归的内存模型：栈（Stack）

理解递归的关键在于理解**栈**。每当函数调用自身时，操作系统都会在内存中开辟一个新的区域（称为“栈帧”）来保存当前函数的局部变量和返回地址。

- **递推（Push）**：函数不断调用自己，栈帧一层层压入。
    
- **回归（Pop）**：触碰到终止条件后，函数开始返回，栈帧一层层弹出并释放。
    

---

## 3. 经典案例：计算阶乘 ($n!$)

阶乘的数学定义本身就是递归的：

- $0! = 1$ （基准情形）
    
- $n! = n \times (n-1)!$ （递归步骤）
    

### C 代码实现

C

```
#include <stdio.h>

// 递归函数计算阶乘
long factorial(int n) {
    // 1. 基准情形：如果 n 是 0 或 1，直接返回 1
    if (n <= 1) {
        return 1;
    }
    // 2. 递归步骤：n * (n-1) 的阶乘
    else {
        return n * factorial(n - 1);
    }
}

int main() {
    int num = 5;
    printf("%d 的阶乘是: %ld\n", num, factorial(num));
    return 0;
}
```

## 4. 递归 vs 循环（Iteration）

|**维度**|**递归**|**循环 (for/while)**|
|---|---|---|
|**代码结构**|简洁、优雅，更接近数学定义|相对冗长|
|**性能**|较慢（频繁的函数调用开销）|快（直接跳转指令）|
|**内存占用**|高（每一层都要占用栈空间）|低（固定占用）|
|**风险**|**栈溢出 (Stack Overflow)**|死循环（通常不崩溃内存）|

---

## 5. 什么时候使用递归？

虽然循环通常更高效，但在处理以下场景时，递归的代码逻辑会清晰得多：

1. **分治算法**（如：快速排序、归并排序）。
    
2. **数据结构遍历**（如：树的遍历、图的搜索）。
    
3. **复杂数学问题**（如：汉诺塔、斐波那契数列）。
    

---

## 6. 避坑指南：栈溢出 (Stack Overflow)

如果你忘记写终止条件，或者递归层数太深（例如递归 100 万次），你的程序会耗尽操作系统分配给它的栈内存，导致程序崩溃。

> **技巧**：在嵌入式开发（你目前的领域）中，由于硬件内存（RAM）非常有限，**应谨慎使用深度递归**。如果能用 `while` 循环解决，通常优先选择循环。
