## 1.求模运算符（%）
用于整数运算，**用于计算两个整数相除后的余数**。
- `10 % 3` 结果为 `1` （10除以3商3余1）
    
- `12 % 4` 结果为 `0` （12能被4整除）
    
- `3 % 10` 结果为 `3` （3除以10商0余3）
C语言规定求模运算的结果符号与**被除数 (a)** 的符号保持一致，只能用于整数运算。
### 2.递增递减运算符（++  --）
① 前缀模式 (`++i`, `--i`)

**规则：先变值，后使用。** 程序会先对变量进行加1（或减1），然后再把改变后的值带入到当前的表达式中计算。

 ② 后缀模式 (`i++`, `i--`)

**规则：先使用，后变值。** 程序会先取变量当前的值用于表达式计算，等包含该运算符的语句执行完毕后，再对变量进行加1（或减1）。

```
int a = 5, b = 5;
int x, y;

x = ++a;  // 前缀：a先变成6，然后赋值给x。结果：x=6, a=6
y = b++;  // 后缀：先将b的值(5)赋值给y，然后b再变成6。结果：y=5, b=6
```
## 3.逗号运算符
在C语言中，**逗号运算符（Comma Operator）** 是优先级**最低**的运算符。它的存在主要是为了将多个表达式组合在一起，并按照从左到右的顺序依次执行。
1. 基本语法与规则
逗号表达式的一般形式为：

$$表达式1, 表达式2, 表达式3, \dots, 表达式n$$

- **执行顺序**：从左向右依次计算每个表达式。
    
- **最终结果**：整个逗号表达式的值是**最后一个表达式（表达式n）**的值。
    
- **类型**：整个表达式的类型也是最后一个表达式的类型。
## 2. 代码示例

### 场景一：基本赋值

C

```
int a, b;
a = (b = 3, b + 2); 
```

- **执行过程**：
    
    1. 先计算 `b = 3`，此时 `b` 的值变为 3。
        
    2. 再计算 `b + 2`，结果为 5。
        
- **结果**：变量 `a` 最终被赋值为 **5**。
    

### 场景二：在 for 循环中（最常用）

逗号运算符最常见的舞台是在 `for` 循环的初始化或更新部分，允许你同时操作多个变量。

C

```
for (int i = 0, j = 10; i < j; i++, j--) {
    printf("i=%d, j=%d\n", i, j);
}
```

- 这里 `i++, j--` 就是一个逗号表达式，每次循环结束时，两个变量都会更新。

## 3. 优先级陷阱：为什么要加括号？

逗号运算符的优先级是**全C语言最低**的，甚至比赋值运算符 (`=`) 还要低。

- **没有括号的情况：**
    
    C
    
    ```
    int x;
    x = 1, 2, 3; 
    // 等价于 (x = 1), 2, 3;
    // 最终 x 的值是 1。
    ```
    
- **有括号的情况：**
    
    C
    
    ```
    int x;
    x = (1, 2, 3);
    // 先计算括号内的逗号表达式，结果为 3。
    // 最终 x 的值是 3。
    ```

## 4. 易混淆点：是“运算符”还是“分隔符”？

并不是程序中出现的每个逗号都是“逗号运算符”。在以下情况下，逗号仅作为**分隔符**使用：

1. **变量声明**：`int a, b, c;` （这只是分隔变量名）
    
2. **函数参数**：`printf("%d %d", a, b);` （这只是分隔不同的参数）
    

**区别方法**：看它是否在一个可以放置独立表达式的地方。如果是函数调用 `func(a, (b, c))`，那么 `a` 和 `(b, c)` 之间的是分隔符，而 `(b, c)` 内部的是逗号运算符。

## 5. 在嵌入式开发中的应用建议

在你关注的**嵌入式/C语言底层开发**中，逗号运算符有时会出现在宏定义（Macro）中，用来确保一系列操作能像单个表达式一样被执行并返回一个值。

但由于它会降低代码的可读性，==**除非在 `for` 循环中，否则不建议过度使用逗号运算符**==。清晰的代码结构（分行写）通常比精简的单行逗号表达式更好维护。
