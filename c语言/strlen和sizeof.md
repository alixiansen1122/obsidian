
在 C/C++ 编程中，`sizeof` 和 `strlen` 是两个经常被混淆但在底层逻辑上完全不同的概念。简单来说，**`sizeof` 是“称重”，而 `strlen` 是“数数”**。
### 1. 基本定义与性质

|**特性**|**sizeof**|**strlen**|
|---|---|---|
|**本质**|**运算符** (Operator)|**标准库函数** (`string.h`)|
|**计算时间**|**编译时** (Compile-time) 计算|**运行时** (Runtime) 计算|
|**计算对象**|类型、变量、数组、指针等所占的**内存字节数**|以 `\0` 结尾的字符串的**有效长度**|
|**包含 `\0` 吗**|包含（如果计算的是字符数组）|**不包含**|
sizeof的返回值类型为size_t
---

### 2. 核心差异详解

#### A. 作用对象不同

- **`sizeof`**：可以作用于任何数据类型（int, double, 指针, 结构体）或变量。它告诉你该对象在内存中占据了多少空间。
    
- **`strlen`**：只能作用于以 `\0` (Null terminator) 结尾的**字符指针或数组**。它会从起始地址开始遍历，直到遇到第一个 `\0` 为止。
#### B. 数组与指针的陷阱

这是最容易出错的地方。请看下面的代码示例：

C

```
char arr[10] = "Hello";
char *ptr = "Hello";

// 对于数组 arr
printf("%zu", sizeof(arr));   // 输出 10 (整个数组的大小)
printf("%zu", strlen(arr));   // 输出 5  (H-e-l-l-o 这5个字符)

// 对于指针 ptr
printf("%zu", sizeof(ptr));   // 输出 4 或 8 (指针变量本身的大小，取决于系统位数)
printf("%zu", strlen(ptr));   // 输出 5  (字符串的逻辑长度)
```

#### C. 计算时机

- **`sizeof`** 在程序还没运行（编译阶段）就已经确定好值了。所以你不能用它来获取动态分配内存（如 `malloc`）的实际内容长度。
    
- **`strlen`** 必须在程序运行时通过扫描内存来确定长度。
    

---

### 3. 常见对比总结

|**场景**|**sizeof 的表现**|**strlen 的表现**|
|---|---|---|
|`char str[20] = "Hi";`|**20** (数组定义的总容量)|**2** (有效字符数)|
|`char *p = "Hi";`|**4 或 8** (指针的大小)|**2** (有效字符数)|
|`int a[5];`|**20** (5个int * 4字节)|**编译报错** (参数必须是字符型)|
|遇到 `\0`|将其视为内存的一部分继续计算|**立即停止**，返回之前的计数|

---

### 4. 避坑指南

1. **不要对指针使用 `sizeof` 来获取字符串长度**：它只会返回指针本身的固定大小。
    
2. **缓冲区溢出**：使用 `strlen` 时，必须确保字符串以 `\0` 结尾，否则它会继续向后读取，直到发生崩溃或读取到随机值。
    
3. **性能消耗**：`strlen` 需要遍历字符串，时间复杂度是 $O(n)$；而 `sizeof` 是常量值，性能开销为 0。
