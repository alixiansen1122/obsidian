# C 语言重定向 (Redirection) 详解笔记

## 1. 核心概念：标准流 (Standard Streams)

在操作系统中，每个进程在启动时都会默认打开三个文件流。重定向的本质就是改变这些流的**源（Source）或去向（Destination）**。

|**流名称**|**文件描述符**|**默认设备**|**C 指针**|**说明**|
|---|---|---|---|---|
|**标准输入 (stdin)**|0|键盘|`stdin`|程序读取输入的地方|
|**标准输出 (stdout)**|1|终端屏幕|`stdout`|程序打印正常信息的地方|
|**标准错误 (stderr)**|2|终端屏幕|`stderr`|程序打印错误诊断信息的地方|

## 2. 命令行重定向 (Shell Redirection)

这种方式不需要修改代码，直接在终端/命令行执行程序时通过符号控制。

假设编译后的程序名为 demo.exe (Windows) 或 ./demo (Linux)。

### 2.1 输入重定向 `<`

让程序从文件读取内容，而不是等待键盘输入。

- **语法**：`程序 < 输入文件`
    
- **示例**：`./demo < data.txt` （程序会自动读取 data.txt 中的字符）
    

### 2.2 输出重定向 `>` 和 `>>`

将程序原本要显示在屏幕上的内容保存到文件中。

- **覆盖写入 `>`**：`./demo > output.txt` （清空并写入，若文件不存在则创建）
    
- **追加写入 `>>`**：`./demo >> log.txt` （在文件末尾继续写，不破坏原有内容）
    

### 2.3 错误重定向 `2>`

专门将报错信息定向到文件，而不干扰正常输出。

- **示例**：`./demo 2> error.log`
    

### 2.4 管道 `|`

将前一个程序的输出直接作为下一个程序的输入。

- **示例**：`cat input.txt | ./demo`

## 3. 程序内重定向：freopen() 函数

如果你希望程序运行后自动关联文件，而不依赖命令行操作，可以使用 `freopen`。

### 3.1 函数原型

C

```
FILE *freopen(const char *filename, const char *mode, FILE *stream);
```

- **filename**: 目标文件名。
    
- **mode**: 访问模式（`"r"` 读取, `"w"` 写入, `"a"` 追加）。
    
- **stream**: 要重定向的流（`stdin`, `stdout` 或 `stderr`）。
    

### 3.2 示例代码

C

```
#include <stdio.h>

int main() {
    // 1. 将输入重定向到 in.txt (后续 getchar, scanf 都会从该文件读取)
    freopen("in.txt", "r", stdin);

    // 2. 将输出重定向到 out.txt (后续 putchar, printf 都会写入该文件)
    freopen("out.txt", "w", stdout);

    int ch;
    while ((ch = getchar()) != EOF) {
        putchar(ch);
    }

    return 0;
}
```

## 4. 重定向的实际意义

1. **自动化测试**：通过输入重定向大量测试用例，无需手动输入，提高效率。
    
2. **日志记录**：将程序的运行状态或错误记录到 `.log` 文件中，方便后续排查。
    
3. **数据持久化**：将计算结果直接保存为文件，供其他工具（如 Excel、Python）处理。
    
4. **静默运行**：在后台运行时，将输出重定向到 `/dev/null` (Linux) 或 `NUL` (Windows) 以隐藏所有提示。
    

---

## 5. 注意事项

- **EOF 识别**：重定向到文件后，当读取到文件末尾时，`getchar()` 会自动触发 `EOF`。
    
- **路径问题**：使用 `freopen` 时，确保文件路径正确，否则会返回 `NULL`。
    
- **恢复标准流**：在某些复杂的应用中，重定向到文件后可能需要恢复到屏幕，但在标准 C 中，恢复 `stdin/stdout` 比较复杂且依赖平台。通常建议只在整个程序生命周期内使用重定向。
