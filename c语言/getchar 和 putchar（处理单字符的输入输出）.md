`<stdio.h>` 头文件中
## 1. getchar()：获取字符

`getchar` 的功能是从标准输入（通常是键盘）读取**一个**字符。

### 函数原型

```
int getchar(void);
```
### 关键特性

- **返回值类型是 `int`**：虽然它读取的是字符，但返回值是 `int` 型。这是为了能够返回一个特殊值 **EOF**（End Of File，通常是 -1），表示读取结束或出错。
    
- **阻塞式等待**：当程序运行到 `getchar()` 时，如果输入缓冲区没有内容，程序会停下来等待用户输入。
    
- **行缓冲机制**：这是初学者最容易困惑的地方。`getchar` 并不是你按下一个键就立刻读取，而是等到你按下 **回车键（Enter）** 后，整个字符串被送入缓冲区，`getchar` 才从缓冲区中逐个提取字符。
    
## 2. putchar()：输出字符

`putchar` 的功能是将**一个**字符输出到标准输出（通常是屏幕）。
### 函数原型

```
int putchar(int char);
```
### 关键特性

- **参数**：它可以接收一个字符变量、一个字符常量（如 `'A'`），或者一个整数（ASCII 码）。
    
- **返回值**：如果执行成功，返回写入的字符；如果失败，返回 `EOF`。
    
## 3. 核心区别与协作

我们可以通过一个简单的表格来对比

|**特性**|**getchar**|**putchar**|
|---|---|---|
|**功能**|从键盘读取一个字符|向屏幕打印一个字符|
|**参数**|无|要输出的字符（或其 ASCII 码）|
|**返回值**|读取到的字符（int）或 EOF|输出的字符（int）或 EOF|

### 典型应用示例：字符回显

下面的代码会把你输入的每一个字符实时打印出来，直到你输入一个特定的结束标志（在 Windows 上通常是 `Ctrl+Z`，Linux/macOS 上是 `Ctrl+D`）。

```
#include <stdio.h>

int main() {
    int c;

    printf("请输入一些字符（按 Ctrl+D/Z 结束）：\n");

    // 当读取的字符不是文件结束符 EOF 时，循环继续
    while ((c = getchar()) != EOF) {
        putchar(c);
    }

    return 0;
}
```

## 4. 进阶小贴士：那个讨厌的“回车符”

在使用 `getchar()` 时，大家常遇到一个坑：**回车键本身也是一个字符（`\n`）**。

如果你写了类似下面的代码：

```
char a, b;
a = getchar();
b = getchar();
```

当你输入 `A` 然后按回车时，变量 `a` 会得到 `'A'`，而变量 `b` 会自动得到 `'\n'`（换行符），导致你无法输入第二个字母。这就是为什么在处理连续输入时，经常需要用一个额外的 `getchar()` 来“吃掉”多余的回车符。
