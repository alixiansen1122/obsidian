#### 通俗解释
通常的 AT 指令交互是“一问一答”：
- **MCU 说：** “现在的信号怎么样？” (`AT+CSQ`)
- **模组 说：** “信号 25。” (`+CSQ: 25`) -> **这是标准响应 (Response)**。
但在某些时候，模组发生了一些突发事件（比如有人给你打电话，或者网络断了），模组不能等你来问，必须立刻喊你。
- **模组 突然说：** “有电话来了！” (`RING`) -> **这就是 URC**。
- **模组 突然说：** “收到短信了！” (`+CMTI: "SM", 1`) -> **这就是 URC**。
 #### 技术定义

URC 是通信模组在**没有收到 MCU 发出的 AT 指令**的情况下，主动通过串口（UART）向 MCU 发送的字符串信息。它通常用于通知 MCU 发生了异步事件。

### 2. 什么是“注册 URC”？
**注册 URC** (Registration URC) 是 URC 的一种特定类型，专门用于**汇报网络注册状态的变化**。
#### 核心作用
它解决了一个痛点：**设备如果是移动的，或者信号不稳定，MCU 怎么知道什么时候断网了？** 如果不开启注册 URC，MCU 就必须每隔几秒发送一次 `AT+CREG?` 去轮询查询，这既浪费 CPU 资源，又占用串口通道。
开启注册 URC 后，模组会在状态改变的瞬间通知你：
- 从“有网”变成“没网” -> 模组立刻吐出 `+CREG: 0`。
- 从“没网”变成“有网” -> 模组立刻吐出 `+CREG: 1`。
#### 如何开启？

注册 URC 默认通常是关闭的，需要你用指令开启（这就解释了为什么你的初始化代码很重要）：

- **`AT+CREG=0`**：**关闭 URC**。模组变成哑巴，网络断了也不告诉你。你只能通过 `AT+CREG?` 主动去查。
    
- **`AT+CREG=1`**：**开启简单 URC**。网络变动时，模组会上报 `+CREG: <stat>`（只告诉你状态变了）。
    
- **`AT+CREG=2`**：**开启详细 URC**。网络变动时，模组会上报 `+CREG: <stat>, <lac>, <ci>`（不仅告诉你状态，还告诉你连到了哪个基站）。
